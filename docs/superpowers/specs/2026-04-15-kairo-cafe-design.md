# 开罗创意咖啡店攻略 - 设计文档

## 概述

为 MyTools 添加"开罗创意咖啡店攻略"工具，帮助玩家追踪顾客烦恼任务和菜谱收集进度。

## 技术架构

### 目录结构
```
backend/tools/kairo-cafe/
├── config.py          # 工具配置
├── handler.py         # 数据加载、持久化、API
├── templates/
│   └── index.html     # 前端页面
└── data/              # JSON缓存目录（自动生成）
```

### 数据存储

**JSON缓存路径：** `backend/tools/kairo-cafe/data/`

- `customer_complaints.json` - 顾客烦恼数据
- `recipes.json` - 菜谱大全数据

**数据结构：**

顾客烦恼（customer_complaints.json）：
```json
{
  "序号": 1,
  "顾客": "主妇",
  "名称": "黄油吐司的饮品",
  "条件": "选：咖啡、冰咖啡、冰红茶",
  "报酬": "菜谱：黄油吐司",
  "是否已完成": false
}
```

菜谱大全（recipes.json）：
```json
{
  "序号": 1,
  "饮品": "冰咖啡",
  "类别": "冰咖啡",
  "食材1": null,
  "食材2": null,
  "装饰物": null,
  "价格": "620G",
  "是否已完成": false
}
```

首次请求时从Excel读取并生成JSON缓存，后续直接读取JSON。

## API接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `get_customer_complaints` | GET | 获取顾客烦恼列表 |
| `get_recipes` | GET | 获取菜谱大全列表 |
| `toggle_complaint_complete` | POST | 切换顾客烦恼状态，body: `{index: int}` |
| `toggle_recipe_complete` | POST | 切换菜谱状态，body: `{index: int}` |

## 前端页面

### 布局
- Tab切换布局：顶部两个Tab "顾客烦恼" | "菜谱大全"
- 每个Tab独立搜索栏
- 表格显示对应列

### 顾客烦恼Tab
| 列 | 说明 |
|----|------|
| 序号 | 编号 |
| 顾客 | 顾客名称 |
| 名称 | 任务名称 |
| 条件 | 饮品选项（饮品名称可点击跳转菜谱大全） |
| 报酬 | 奖励说明 |
| 操作 | 切换完成状态按钮 |

**条件列饮品链接：**
- 识别条件中的饮品名称（逗号分隔）
- 每个饮品名称做成可点击链接
- 点击跳转到"菜谱大全"Tab，自动填入搜索词并搜索该饮品
- 显示回退按钮，点击返回顾客烦恼Tab

### 菜谱大全Tab
| 列 | 说明 |
|----|------|
| 序号 | 编号 |
| 饮品 | 饮品名称 |
| 类别 | 分类 |
| 食材1 | 主食材 |
| 食材2 | 副食材 |
| 装饰物 | 装饰 |
| 价格 | 价格 |
| 操作 | 切换完成状态按钮 |

### 搜索功能
- 支持模糊搜索
- 顾客烦恼：按"名称"搜索
- 菜谱大全：按"饮品"搜索

## 实现步骤

1. 创建目录 `backend/tools/kairo-cafe/`
2. 编写 `config.py`
3. 编写 `handler.py`（数据加载、JSON持久化、API）
4. 编写 `templates/index.html`（前端页面）
5. 编写 Playwright 测试用例
